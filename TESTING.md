# Тестирование плагина ASP Classic Language Support

## Быстрая проверка

### 1. Сборка и запуск

```bash
# Собрать плагин
./gradlew clean build

# Запустить IDE с плагином
./gradlew runIde
```

### 2. Тестирование функциональности

1. **Открыть test-example.asp** в запущенной IDE
2. **Проверить подсветку синтаксиса:**
   - HTML теги должны подсвечиваться
   - Атрибуты должны иметь другой цвет
   - CSS внутри `<style>` должен подсвечиваться

3. **Проверить автодополнение:**
   - Начать печатать `<di` и нажать Ctrl+Space (Cmd+Space на Mac)
   - Должны появиться подсказки HTML тегов (div, dir, etc.)

4. **Проверить навигацию:**
   - Кликнуть на открывающий тег `<div>`
   - Должен подсветиться соответствующий закрывающий тег `</div>`

5. **Проверить валидацию:**
   - Создать неправильный HTML (например, `<div>test`)
   - Должна появиться ошибка об отсутствии закрывающего тега

6. **Проверить форматирование:**
   - Испортить форматирование (убрать отступы)
   - Нажать Ctrl+Alt+L (Cmd+Option+L на Mac)
   - Код должен отформатироваться

### 3. Проверка расширений файлов

Создать файлы и проверить, что они открываются с правильной иконкой:
- `test.asp` - должен показывать ASP иконку
- `global.asa` - должен показывать ASP иконку

## Текущие возможности

✅ **Работает:**
- Подсветка HTML синтаксиса
- Автодополнение HTML тегов и атрибутов  
- Валидация HTML
- Навигация по тегам
- Code folding
- Форматирование кода
- Все HTML фичи IDE

❌ **Не работает (как и задумано):**
- Подсветка VBScript кода внутри `<% %>`
- Автодополнение ASP объектов
- Парсинг ASP директив

Нет известных проблем с текущей реализацией.

## Структура проекта (минимальная)

```
src/main/kotlin/com/dmitry/aspclassic/
├── AspLanguage.kt           # Определение языка ASP
├── AspFileType.kt           # Регистрация типа файла
├── AspIcons.kt              # Иконка
├── AspParserDefinition.kt   # Делегирование к HTML парсеру
└── psi/
    └── AspFile.kt           # PSI файл
```

## Логи и отладка

Если что-то не работает, проверьте:

1. **Логи IDE:**
```bash
tail -f build/idea-sandbox/PS-2024.3.1.1/log/idea.log
```

2. **Проверить, что плагин загружен:**
   - Settings → Plugins → Installed
   - Должен быть "ASP Classic Language Support"

3. **Проверить файл ассоциируется с правильным типом:**
   - Открыть .asp файл
   - Внизу справа должно быть "ASP" (тип файла)
